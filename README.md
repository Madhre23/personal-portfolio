Part 2: Design
**1. What overall style will best represent you?**

A clean, modern, and professional style best represents me. Since the website is focused on technology and IT-related roles, I want it to look organized, readable, and professional while still showing personality.

**2. What color scheme will you use and why?**

I will use a light blue background, dark navy text, and a pink accent color. The light blue gives the site a calm and clean look, the dark navy keeps the text professional and readable, and the pink accent matches the avatar used in the hero section. The pink is used for buttons, underlines, icons, borders, and hover effects to keep the design consistent.

**3. What fonts will you use for headings and body text?**

I will use the Montserrat font from Google Fonts for both headings and body text. Headings will use bold font weights to stand out, while body text will use lighter weights for readability.

**4. How will your design reflect your personality or field?**

The design reflects my field by using a structured layout, clean sections, technical icons, project cards, and professional colors. The avatar and accent color add personality, while the content focuses on IT support, systems administration, and technical growth.

**5. What layout will your homepage follow?**

The homepage will use a hero section layout. It includes a fixed navigation bar, a GM logo, a cartoon avatar, my name, my title as a Computer Science student, a short introduction, buttons to view projects or contact me, and social icons for GitHub, LinkedIn, and email.

**6. How will you organize project sections visually?**

Projects will be organized using card-style sections. Each project card includes the project title, technologies used, a short description, and any relevant media or links. The East West Law Office project includes a clickable video thumbnail that opens the demo video.

**7. Will the site be mobile-friendly? How will you ensure responsiveness?**

Yes, the site will be mobile-friendly. I will use responsive CSS, flexible grid layouts, media queries, and a mobile-friendly navigation menu. On smaller screens, the skill cards and project cards will stack vertically so the website remains easy to read and use.

**8. What visual hierarchy will guide visitors?**

The visual hierarchy will be guided by large section headings, pink underlines, bold titles, card layouts, buttons, icons, and spacing. The most important information appears first in each section, and the buttons guide visitors toward viewing projects, downloading the resume, or contacting me.

**9. How will consistency be maintained across pages?**

Since the website is a single-page portfolio, consistency will be maintained through repeated design patterns. All sections use the same background color, heading style, pink underline, dark navy text, rounded cards, hover effects, and spacing.

**10. How will accessibility be considered?**

Accessibility will be considered by using readable font sizes, strong color contrast, clear navigation labels, descriptive alt text for images, large clickable buttons, and a responsive layout. The website also uses semantic HTML sections to keep the structure organized.

**11. Will you use icons, images, or illustrations? Why?**

Yes, I will use icons, images, and illustrations. The hero section includes a cartoon avatar to add personality. The technical skills section uses icons to represent each skill category. The volunteering section uses a location icon, and the contact section uses GitHub, LinkedIn, and email icons. Part 3: Interactivity
**1. What interactive elements will your site include?**

The site will include navigation links, buttons, hover effects, social media icons, clickable project demo thumbnail, a resume download button, and email contact links.

**2. Will your site include a contact form? How will it work?**

Instead of a full contact form, my site will use direct contact links. Visitors can click the email icon or the “Say Hi” button to open an email message. This works better for a static GitHub Pages website because it does not require a backend or database.

**3. What JavaScript features will you implement?**

The JavaScript will support the mobile navigation menu and scroll-based section transition effects. These features help the site feel smoother and more interactive as users move through the page.

**4. How will users receive feedback from interactions?**

Users will receive feedback through hover effects on buttons, cards, icons, and links. For example, skill cards reveal more information when hovered over, project cards lift slightly, buttons change color, and icons respond visually when users interact with them.

**5. How does interactivity improve the user experience?**

Interactivity improves the user experience by making the website feel more responsive, polished, and engaging. It helps visitors understand what they can click, makes the content easier to explore, and creates a smoother experience while navigating the portfolio.
